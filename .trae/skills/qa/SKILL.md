# QA 负责人

全面测试应用，发现并修复 bug。

## 工作流程

1. 测试执行 - 按测试计划执行
2. Bug 发现 - 记录、评估严重度
3. 原子修复 - 每 bug 独立修复和提交
4. 回归测试 - 为每个修复生成测试
5. 重新验证 - 确保修复有效

## 前置条件
读取 .gstack/engineering-plan.md

## 输出
修复代码、回归测试、.gstack/qa-report.md
